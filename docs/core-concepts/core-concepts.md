# Core Concepts

## Protocol

Represents an onboarded infrastructure provider with:

- Owner wallet identity
- Protocol metadata
- Compensation pool balance
- Reliability commitments
- Incident history

## Commitment

A formal promise made by a protocol that can later be evaluated and verified.

Commitments include supporting evidence and fulfillment evaluation.

## Incident

A formal reliability or security case tracked through a defined lifecycle:

1. **Candidate**: incident is submitted and under review
2. **Challenge Open**: dispute window is active
3. **Finalized**: incident outcome is confirmed
4. **Payout Execution**: compensation is distributed

## Event Verification

CertLayer verifies whether a specific condition has occurred using intelligent contracts and validator verification.

Examples include:

- Protocol downtime
- Oracle price feed failure
- Bridge security incidents
- Infrastructure outages

## Compensation Pool

A protocol-funded balance used to execute automated compensation payments when incidents are finalized.

## Reliability Monitoring

CertLayer creates a transparent infrastructure monitoring system where protocols can publicly demonstrate operational reliability.

## Reputation

A public summary derived from incident outcomes and operational performance, accessible through the CERTLAYER public explorer.

## Security Signal

A suspicious transaction analysis event generated by HackDetection.

Signals include:

- Transaction risk score
- Detected attack patterns
- Potential exploit indicators

## Emergency Pause

A pause signal emitted by HackDetection when suspicious activity is detected.

:::note
A protocol is only paused if integration exists on the protocol side. Integration may involve on-chain guard checks or off-chain automation bots.
:::

## Risk Scoring

Each monitored transaction hash receives a dynamic risk score based on detected patterns.

Example signals include:

- Abnormal contract interactions
- Rapid liquidity withdrawals
- Exploit-like transaction structures
- Interactions with known malicious addresses

Risk scores allow protocols and monitoring systems to react before damage escalates.

## Emergency Response Controls

HackDetection supports emergency response tools including:

- Circuit-breaker pause signals
- Address blacklisting
- Security event records
- Automated monitoring integrations

These controls allow rapid mitigation during suspected exploits.
